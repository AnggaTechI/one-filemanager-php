## 📖 Overview

**one-filemanager-php** is a lightweight, single-file PHP file manager designed for quick deployment and simple web-based file management.

Built to be portable and easy to use, this project can run on environments ranging from **PHP 5.4 up to the latest PHP versions**.

It is especially useful for:

- Rapid deployment on shared hosting or VPS environments
- Emergency file access and maintenance
- Lightweight file operations without FTP clients
- Simple server-side file browsing and editing

## ✨ Features

- Single-file deployment
- No external dependencies
- File and directory browsing
- File upload and download
- File and folder rename
- File and folder deletion
- Create new directories
- Edit text-based files directly from the browser
- Simple and lightweight interface
- Broad compatibility across PHP versions
- Portable and easy to move between servers

## 🧩 Why Use This Project?

This project is intended for users who need a straightforward way to manage files and directories directly from a browser without installing a full application stack or additional dependencies.

If you need something that is:

- fast to upload,
- easy to run,
- simple to maintain,
- and useful in emergency access scenarios,

then **one-filemanager-php** is a practical solution.

---

## ⚙️ Requirements

Before using this project, make sure your environment has:

- **PHP 5.4 or newer**
- A web server such as **Apache**, **Nginx**, or **OpenLiteSpeed**
- Proper **read/write permissions** for the target directories

---

## 🧪 Compatibility

This project supports:

- **PHP 5.4**
- PHP 5.5+
- PHP 7.x
- PHP 8.x
- **Latest stable PHP releases**

---

## 🚀 Installation

1. Upload the PHP file to your server.
2. Place it inside the desired web-accessible directory.
3. Open it through your browser.

Example:

```bash
https://yourdomain.com/one-filemanager-php.php
```

## 🛡️ Security Notice

Because this script provides direct access to file management through the browser, proper security precautions are strongly recommended.

### Recommended security practices

- Change the default password immediately
- Restrict access by IP, HTTP authentication, or additional server-side protection
- Avoid exposing the file manager publicly unless necessary
- Remove the file manager when it is no longer needed
- Ensure directory permissions are configured properly
